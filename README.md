# AI Router & Memory Pipelines for Open WebUI

Набор плагинов для Open WebUI, обеспечивающий интеллектуальную маршрутизацию запросов и управление долговременной памятью пользователя.

## Возможности

- Интеллектуальная маршрутизация: автоматическая классификация запросов (чат, генерация изображений, анализ фото, поиск, сложные вычисления, работа с веб-страницами и файлами) с роутингом на специализированные модели.
- Семантическая классификация: быстрый анализ интентов через эмбеддинги с кэшированием и fallback на LLM.
- Долговременная память: автоматическое извлечение фактов из диалогов, сохранение в Redis с TTL и дедупликацией, инъекция контекста в системный промпт.
- Поиск в интернете: интеграция с SearXNG, автоматический парсинг и форматирование результатов.
- Поддержка потоковой передачи ответов (SSE).

## Требования

- Docker и Docker Compose
- Доступ к OpenAI-совместимому API
- ~2 ГБ свободной оперативной памяти

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone <repository-url>
   cd <project-directory>
   ```

2. Настройте окружение:
   ```bash
   cp .env_example .env
   # Отредактируйте .env — укажите реальные ключи (см. комментарии в .env_example)
   ```

3. Запустите сервисы:
   ```bash
   docker compose up -d
   ```

4. Откройте интерфейс Open WebUI: `http://localhost:3000`

## Конфигурация

Скопируйте `.env_example` в `.env` и заполните значения:

| Переменная | Описание |
|------------|----------|
| `MWS_API_KEY` | Ключ доступа к API моделей (обязательно) |
| `MWS_BASE_URL` | Базовый URL API (по умолчанию `https://api.gpt.mws.ru/v1`) |
| `PIPELINES_API_KEY` | Ключ для доступа Open WebUI к сервису pipelines |
| `WEBUI_SECRET_KEY` | Секретный ключ сессий Open WebUI |
| `SEARXNG_SECRET_KEY` | Секрет SearXNG (`openssl rand -hex 32`) |
| `FORCE_REASONING` | `true` — принудительно модель рассуждений |
| `REDIS_HOST` / `REDIS_PORT` | Подключение к Redis (по умолчанию `hackathon-redis:6379`) |

Параметры пайплайнов (модели, TTL памяти, лимит фактов) настраиваются в классе `Valves` внутри `pipelines/router.py` и `pipelines/memory.py`.

## Использование

Система работает автоматически. Примеры запросов:
- `Нарисуй закат над морем` -> генерация изображения
- `Какой курс доллара сегодня?` -> поиск в интернете
- `Проанализируй https://example.com` -> загрузка и анализ веб-страницы
- `Сделай саммари файла` -> обработка документа

Для явного указания маршрута используйте префиксы:
- `[SEARCH] ваш запрос`
- `[IMAGE_GEN] ваш запрос`
- `[WEB] ваш запрос`

Память обновляется автоматически после каждого ответа.

## Архитектура

Стек запускается через Docker Compose:
- `open-webui`: интерфейс чата (`:3000`)
- `pipelines`: маршрутизация и память (`:9099`)
- `redis`: хранилище фактов о пользователях
- `searxng`: поисковый движок (`:8080`)

## Примечания

- Файл `.env` в `.gitignore` — не коммитьте его в репозиторий.
- Перед push: `./scripts/check-secrets.sh`
- Кэш эмбеддингов: `pipelines/embeddings_cache.json` (ускоряет повторный запуск).
- Логи: `docker compose logs -f pipelines`

3. Обновите GitHub переписанной историей (осторожно, переписывает `main`):
