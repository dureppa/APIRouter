# Безопасность

## Секреты

- Никогда не коммитьте файл `.env` — он в `.gitignore`.
- Используйте `.env_example` как шаблон: `cp .env_example .env` и заполните значения локально.
- После клонирования сгенерируйте новые ключи, не переиспользуйте ключи из примеров.

## Ротация ключей (если репозиторий был публичным со старыми коммитами)

Если в истории Git когда-либо попадали реальные ключи, **считайте их скомпрометированными** и замените:

| Секрет | Где заменить |
|--------|----------------|
| `MWS_API_KEY` | Личный кабинет / админка MWS API |
| `PIPELINES_API_KEY` | Новое значение в `.env` (любая длинная случайная строка) |
| `WEBUI_SECRET_KEY` | Новый ключ в `.env` |
| `SEARXNG_SECRET_KEY` | `openssl rand -hex 32` → `.env` |

## Проверка перед push

```bash
./scripts/check-secrets.sh
```

## Очистка истории Git

Если секреты уже были запушены на GitHub, одного нового коммита недостаточно — нужен **force push** переписанной истории (см. README) и ротация всех ключей выше.
